# 华为备忘录同步 / 网页版保活

## 背景与结论

目标是把华为备忘录用在 Ubuntu 上。调研后的硬约束：

- 华为备忘录**没有官方 API**，没有 WebDAV/CalDAV 之类标准入口；
- **没有 Linux 客户端**（云空间 PC 客户端仅 Windows/Mac）；
- 网页版 `cloud.huawei.com` 能**查看/编辑/搜索/新建**备忘录，但官方明确**不支持批量导出文字内容**（只能逐条选中复制、附件右键另存）。

可行路线对比（曾评估）：

| 方案 | 做法 | 自动化 | 维护 | 结论 |
|---|---|---|---|---|
| A 网页版当客户端 | `cloud.huawei.com` 固定为 Chrome 应用 | 无需 | 零 | **采用** |
| B 定期单向备份 | Selenium/Playwright 抓取转 Markdown + cron | 半自动 | 高（华为登录风控、cookie 过期、页面易改版） | 否 |
| C 一次性整库迁移 | 华为云 GDPR「数据下载」导出加密 zip（含每条笔记 JSON+HTML）→ 解析转 md | 手动+延迟 | 中（一次性） | 备选 |

用户实际诉求 = **继续用已固定在 Chrome 的网页版**，只是它**空闲时很快超时掉登录**、要反复重登。
因此本方案聚焦解决「会话保活」，不重建一套同步。

## 掉线的两个原因

1. **服务端空闲超时**：会话 token 在不操作一段时间后失效（华为系产品典型为分钟级、滑动过期）。
   服务端强制，改不了 —— 但可以**在过期前制造一次已登录请求**来续命。
2. **浏览器未被信任 / cookie 没持久化**：浏览器没勾「信任此设备」时每次新会话要安全验证；
   退出清 cookie 则每次开机要重登。

## 解决方案（三层，本仓库实现 ①②）

### ① 一次性持久化（零风险，先做）

- 登录时勾选**「信任此设备 / 保持登录」**（在安全验证页或登录页）。
- 确认这个 Chrome 应用用的是**会保留 cookie 的常规个人资料**：
  Chrome → 设置 → 隐私和安全 → Cookie → **关闭「关闭所有窗口时清除 Cookie」**；
  或在站点例外里把 `cloud.huawei.com` 设为「可在设备上保留数据」。
- 这一步消除「每次开机重登」。

### ② 保活用户脚本（核心，治空闲超时）

文件：[`huawei-notes-keepalive.user.js`](./huawei-notes-keepalive.user.js)

原理：在**空闲超时到来之前**，自动**悄悄重载页面**。一次重载 = 一次带 cookie 的已登录请求，
把服务端的滑动空闲计时器顶回去，会话就不掉。脚本特性：

- **绝不打断编辑**：检测到光标在输入框/可编辑区、或有文本选中时，跳过本次重载。
- **只在空闲时重载**：默认空闲满 8 分钟才重载（可调）。你在用的时候不会重载。
- **掉线弹桌面通知**：万一保活没顶住、被踢回登录页，弹通知提醒你点一下重登。
- **不存任何密码**。
- 右下角小角标显示「保活中 · N min 后续命」，便于确认它在工作（可关）。

#### 安装步骤

1. 在这个 Chrome 里装 **Tampermonkey** 扩展（Chrome 应用窗口 → 右上角菜单 → 扩展，
   或先在普通 Chrome 装好，同一 profile 的应用窗口会共享）。
2. 打开 Tampermonkey 面板 → 「添加新脚本」/ 直接把 `huawei-notes-keepalive.user.js`
   拖进 Tampermonkey 安装；确认已启用。
3. 打开/重开 `cloud.huawei.com` 备忘录，右下角出现「保活中…」角标即生效。

#### 调参

打开脚本顶部 `可调参数`：

- `RELOAD_AFTER_IDLE_MIN`（默认 8）：**若仍会掉线，调小**（如 5 / 3）。它必须小于华为的空闲超时。
- `SHOW_BADGE`：不想看角标改 `false`。

#### 验证保活是否真的有效（重要）

华为的超时**可能是滑动过期（保活有效）也可能是绝对过期（保活无效）**，需实测一次：

1. 装好脚本，登录备忘录，**放着别动**。
2. 观察是否还会被踢回登录页：
   - 若**几小时都不掉** → 滑动过期，保活成功，完成。
   - 若仍按固定时间掉（比如总在登录后约 X 分钟，无论是否重载）→ 是绝对过期，
     纯客户端续不了命，需要走 ③ 自动重登，或接受偶尔手动重登。

### ③ 自动重登兜底（可选，未实现）

当前用户选择**不做**（不想把账号密码存本地）。如需：脚本在检测到登录页时，
自动填入账号密码并提交（未触发短信/扫码验证时可一步完成）。代价是密码需明文存于本地脚本，
请自行权衡后再加。

## 备选：方案 C 一次性整库导出（需要本地备份时）

`cloud.huawei.com` → 隐私/数据下载（GDPR「下载云空间数据」）→ 申请导出 → 等数小时收到
**加密 zip** → 解压得到 `Notes/`，每条笔记一个文件夹，含 **JSON + HTML**。
社区有把该 JSON/HTML 转 TXT/Markdown 的脚本可参考。适合「搬家 / 整库备份一次」，非实时。
